Abordagens baseadas em Processos-de-Processos de Negócio para apoiar o Desenvolvimento de Sistemas-de-Sistemas
Descrição: Sistemas-de-Sistemas (SoS) estão cada vez mais presentes em vários setores críticos da sociedade, por exemplo, saúde, transporte e defesa. SoS se refere a sistemas de software intensivo grandes e complexos, formados a partir de sistemas constituintes que interagem entre si para realizar missões maiores. A maioria das pesquisas atuais sobre SoS concentra-se em questões técnicas, como projeto arquitetural e garantia de qualidade, mas o desenvolvimento e a evolução de SoS ainda são desafiadores. Ao mesmo tempo, como os sistemas constituintes podem pertencer a diferentes organizações que têm seus processos de negócios específicos, o SoS leva ao surgimento de grandes processos de negócios dinâmicos que abrangem esses processos específicos. Embora esses grandes processos tenham informações essenciais para o desenvolvimento e evolução do SoS, geralmente elas têm sido ignoradas. Este projeto de pesquisa pretende investigar como as informações significativas de grandes processos de negócios dinâmicos podem beneficiar diferentes atividades técnicas para desenvolver e evoluir SoS mais apropriadamente, como Engenharia de Missões de SoS, Engenharia de Requisitos e Verificação, Validação & Testes (VV&T). Algumas pesquisas que nós já temos conduzido, incluindo a proposta do conceito de Processos-de-Processos de Negócio (PoP), ou seja, grandes processos dinâmicos formados a partir de diferentes processos de organizações (que poderiam ser também outros PoPs) e que têm seus sistemas de software como parte de um SoS, apontam que ainda existem muitas oportunidades de pesquisa neste sentido. Como resultado principal, este projeto de pesquisa visa definir abordagens para fornecer visões (representações) adequadas do PoP com o objetivo de apoiar o desenvolvimento e evolução de SoS, bem como extrair informações valiosas dessas visões para mitigar os problemas técnicos existentes durante as fases de todo o ciclo de vida de SoS. Além disso, pretendemos chamar a atenção para a necessidade de considerar o nível dos processos de negócios para melhor lidar com os desafios no nível técnico de SoS (como especificações de missão e interoperabilidade) e, como consequência, promover a sustentabilidade e longevidade do SoS, que são na verdade, impactadas pelo nível de processo..

Abordagem de apoio ao Reúso de Requisitos em Linhas de Processos de Negócio
Descrição: Os modelos de processos de negócio auxiliam a elicitação de requisitos pois facilitam o entendimento do negócio para o qual o software será desenvolvido. Para sistematizar a elicitação de requisitos a partir desse tipo de modelo foram propostos diversos trabalhos, que elicitam requisitos a partir de elementos de notações utilizadas para representar esse tipo de modelo, como é o caso da notação BPMN (Business Process Model and Notation). Porém, esses trabalhos não tratam da extração e reúso de requisitos de artefatos que representam o domínio de processos de negócio, como é o caso das Linhas de Processo de Negócio (LPN), que é uma técnica de reutilização no contexto de modelos de processos de negócio. Na elicitação de requisitos de um determinado domínio de negócio é necessário considerar as suas comunalidades e variabilidades. Outro desafio ocorre durante a reutilização dos requisitos que podem estar duplicados, ou conflitantes, conforme a configuração de variabilidade da linha. Para facilitar a obtenção e a reutilização de requisitos de um determinado domínio de negócio, este trabalho tem como objetivo estabelecer a abordagem ARReq para elicitar requisitos de LPN para permitir a reutilização destes a partir da instanciação destas linhas. Com isso, pretende-se melhorar a eficiência da Engenharia de Requisitos, com a redução do tempo e com o aumento da qualidade dos requisitos obtidos, visto que estarão alinhados com o negócio. Para avaliar a eficácia da ArReq serão realizadas provas de conceitos e estudos de caso utilizando abordagens existentes de gestão de LPNs e suas ferramentas de apoio..

Modelo Arquitetural para o Design de Interfaces Web Acessíveis
Descrição: Projetar interfaces web acessíveis não é uma tarefa trivial pois envolve um conjunto de habilidades e conhecimentos sobre modelos e técnicas de implementação de acessibilidade por parte do projetista. No entanto, não foram encontrados na literatura trabalhos que utilizavam padrões ou estilos arquiteturais para facilitar, reduzir o custo e aumentar a qualidade do projeto de interfaces web acessíveis com enfoque em atender usuários com tipos específicos de deficiência, como é o caso da baixa visão, cegueira, daltonismo, baixo letramento, surdez, idade avançada, dificuldade de aprendizado, motora, intelectural, dentre outros. Sob essa perspectiva, o objetivo deste projeto de pesquisa é definir um modelo arquitetural, baseado no WCAG, que é um modelo de acessibilidade bastante utilizado e difundido, e em padrões e estilos arquiteturais existentes com o intuito de facilitar o projeto de interfaces web acessíveis para diferentes tipos de deficiência. Para isso, a variabilidade de critérios de sucesso que podem ser atendidos, dependendo do nível de acessibilidade do modelo WCAG e do tipo de deficiência, será representada no modelo arquitetural proposto como uma visão arquitetural. Além dessa visão, outras visões arquiteturais serão definidas a fim de atender as diferentes necessidades dos projetistas. Para facilitar o uso do modelo arquitetural, uma ferramenta na forma de plugin será desenvolvida. O modelo arquitetural será avaliado por meio de estudos empíricos com o intuito de avaliar a sua utilidade e a sua capacidade de facilitar o projeto de interfaces web acessíveis para cada tipo de deficiência atendida..

Descrição: O objetivo geral do presente projeto é propiciar a inclusão digital de portadores de necessidades especiais com deficiência mental moderada. Como objetivos específicos têm-se: a) disponibilizar e preparar um laboratório, com softwares educacionais específicos, para uso das aulas de informática aos alunos da APAE, b) viabilizar um aluno voluntário do UNIVEM como monitor para cada três alunos da APAE, e c) acompanhar, orientar, supervisionar, fiscalizar e avaliar a execução e o desenvolvimento do projeto pela professora responsável..

Projeto de extensão
2015 - 2016
SEE: Um aplicativo de Aprendizagem para Deficientes Visuais em Dispositivos Móveis
Descrição: Os Sistemas de Informação podem ser usados como um meio para a democratização do conhecimento e devem viabilizar o acesso a ele para todos os cidadãos sem distinção. Entretanto, grande parte desses sistemas computacionais desenvolvidos não segue os padrões de acessibilidade, o que inviabiliza o acesso à informação por pessoas com deficiência visual. Este projeto visa desenvolver o aplicativo SEE, que poderá ser executado em dispositivos móveis, contendo lições didáticas relacionadas aos níveis de ensino fundamental, médio e superior. O aplicativo será desenvolvido utilizando tecnologias assistivas, a partir de um conjunto de recomendações e diretrizes, que permitem a compreensão do conteúdo pelos deficientes visuais. As lições das disciplinas oferecidas serão apresentadas seguindo as diretrizes de acessibilidade do WCAG (Web Content Accessibility Guidelines), a principal recomendação de acessibilidade a nível internacional. O aplicativo fornecerá conteúdos explicativos das matérias e exercícios, onde os usuários poderão fazer a qualquer hora e em qualquer lugar por estarem disponíveis em dispositivos móveis. Os aplicativos serão desenvolvidos em Java para dispositivos com Android e Objective-C para o iPhone e iPad. O projeto é realizado com conjunto com o Instituto Sul Matogrossense para Cegos Florivaldo Vargas (ISMAC), que oferece atendimento especializado para deficientes visuais inseridos nos Centros de Educação Infantil, escolas municipais, estaduais e particulares (ensino fundamental e médio) e para acadêmicos de diversas Universidades..
