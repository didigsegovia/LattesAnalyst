Fábrica de Jogos Educacionais: definição de domínio e implementação usando aspectos de Linhas de Produto de Software
Descrição: O uso de jogos digitais na educação promove diversos benefícios ao aluno como motivação. persistência além de tornar o ensino mais prazeroso para alunos, e vários jogos com tal finalidade vem sendo desenvolvidos e utilizados em diferentes níveis de ensino entretanto o desenvolvimento de jogos ainda é complexa, principalmente quando se tem viés educativo. As Linha de produto de software (LPS) tem como um dos objetivos corroborar para o desenvolvimento de software reduzindo a complexidade ciclomática, redução do número de linhas de código desenvolvidas pelo programador, redução do tempo para desenvolver e facilidade no desenvolvimento. As LPSs possuem como um dos principais princípios a reutilização de software e se encontra em grande evidência na literatura. LPSs são derivadas da Engenharia de Linhas de Produto, comum na engenharia, e que objetiva customização em massa por meio de produção coletiva, de múltiplas instâncias similares, mas de design distinto. O objetivo deste projeto é definição do domínio de jogos educacionais digitais com a finalidade de ser uma base para uma LPS e para o desenvolvimento de jogos..

Uma Abordagem apoiada Computacionalmente para a Construção, Instanciação e Validação de Linhas de Processos de Negócios
Projeto certificado pelo(a) coordenador(a) Maria Istela Cagnin em 11/07/2014.
Descrição: O entendimento dos processos de negócios de uma empresa pelos seus colaboradores e por engenheiros de software colabora, respectivamente, para o sucesso do negócio e para o desenvolvimento de sistemas de software que atendam às necessidades dos seus interessados. Isso é facilitado pela modelagem de processos de negócios, a qual apoia a análise de requisitos no contexto de Engenharia de Software auxiliando na compreensão e otimização de processos de negócios existentes nas organizações sob a perspectiva de Gestão de Processos de Negócios. Em ambos os contextos, o reúso de modelos de processos de negócios é relevante por propiciar economia de tempo, esforço e custos durante a elaboração de tais modelos. Diante disso, a Engenharia de Linha de Produto de Software (ELPS), tem sido aplicada no contexto de Processos de Negócios, propiciando o desdobramento de outras áreas de pesquisa como é o caso de Linhas de Processos de Negócios (LPN). Nesse contexto, este projeto de pesquisa pretende contribuir para que interessados realizem atividades de modelagem de processos de negócios com menos tempo, esforço e custos associados. Para isso, tem como objetivo refinar uma abordagem existente de linhas de processos de negócios, denominada GLPN (Gestão de Linhas de Processos de Negócio), para atender também a visão de objetivos de negócios, além da visão de processos de negócios já contemplada pela mesma; bem como desenvolver um arcabouço computacional associado a um repositório para apoiar todas as fases da abordagem GLPN relacionadas a criação, validação e instanciação de LPNs. Avaliações qualitativas serão conduzidas para analisar e comprovar a eficiência dos resultados decorrentes deste trabalho..

Descrição: A modelagem de negócios é uma documentação importante para as empresas, pois registra principalmente os processos de negócios, os responsáveis pela sua execução e as regras de negócios envolvidas. Como outros artefatos de software, a modelagem de negócios pode ser reutilizada durante diversas fases do ciclo de vida do software a fim de apoiar no entendimento do mesmo, proporcionando redução de custo e maior produtividade durante as atividades de desenvolvimento e reengenharia de sistemas de informação. No entanto, há carência na literatura de ferramental de apoio para possibilitar tanto a recuperação quanto a instanciação de modelos de negócios existentes para outros projetos, pertencentes a domínios de negócios similares. Nesse contexto, este projeto de pesquisa tem por objetivo estabelecer uma infra-estrutura para possibilitar a reutilização de modelagem de negócio, disponível em diversos domínios, nas fases iniciais do desenvolvimento ou da reengenharia de software. Um repositório baseado em um metamodelo previamente definido deve ser desenvolvido para armazenar as modelagens já realizadas, a fim de possibilitar o reúso dessas em outros projetos diferentes daqueles para os quais foram desenvolvidas. Devem também ser especificados critérios para recuperação dessas informações do repositório pelos interessados. Os modelos recuperados do repositório devem ser instanciados para o contexto do domínio do software que está sendo desenvolvido ou que está sendo submetido à reengenharia. Um processo de instanciação da modelagem de negócio reutilizada deve ser definido, bem como deve ser desenvolvido um apoio computacional para tal processo. Estudos de caso devem ser conduzidos para analisar o uso da infra-estrutura estabelecida, possivelmente, em micro e pequenas empresas..

Fábrica de Jogos Educacionais: definição de domínio e implementação usando aspectos de Linhas de Produto de Software
Descrição: O uso de jogos digitais na educação promove diversos benefícios ao aluno como motivação. persistência além de tornar o ensino mais prazeroso para alunos, e vários jogos com tal finalidade vem sendo desenvolvidos e utilizados em diferentes níveis de ensino entretanto o desenvolvimento de jogos ainda é complexa, principalmente quando se tem viés educativo. As Linha de produto de software (LPS) tem como um dos objetivos corroborar para o desenvolvimento de software reduzindo a complexidade ciclomática, redução do número de linhas de código desenvolvidas pelo programador, redução do tempo para desenvolver e facilidade no desenvolvimento. As LPSs possuem como um dos principais princípios a reutilização de software e se encontra em grande evidência na literatura. LPSs são derivadas da Engenharia de Linhas de Produto, comum na engenharia, e que objetiva customização em massa por meio de produção coletiva, de múltiplas instâncias similares, mas de design distinto. O objetivo deste projeto é definição do domínio de jogos educacionais digitais com a finalidade de ser uma base para uma LPS e para o desenvolvimento de jogos..
